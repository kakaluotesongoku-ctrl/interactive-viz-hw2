项目说明文档 (Submission Documentation)

1. 我们的可视化方案旨在解答什么问题？

本项目旨在通过交互式手段深入探索：“自2014年以来，全球各国家/地区的人均温室气体排放量如何演变？哪些国家是排量‘大户’，而其个体的排放趋势又是否与全球大背景相契合？”

相比于基础的数据展示，本项目进一步探讨了一个深层问题：“可视化设计如何通过视觉操纵（如坐标轴截断）来改读者的判断？” 通过引入“客观模式”与“媒体夸大模式”的切换，本项目旨在引导用户思考数据可视化中的伦理与真相。

2. 设计决策的依据与替代方案

  • 视觉编码选择（Visual Encoding）：
    • 动态排名条形图（Bar Chart Race）： 采用水平条形图展示特定年份排名。我们利用了 ECharts 的 realtimeSort 技术，使柱子在年份切换时能产生丝滑的“超车”动画，直观反映国家间排名的竞争性。
    • 双线面积折线图： 右侧视图不仅展示选中国家的趋势，还创新性地引入了“全球平均基准线”。这为用户提供了一个坐标系，能瞬间判断出该国的排放水平是处于全球领先还是落后。
  • 高级交互技术（Interaction Techniques）：
    • 自动播放（Auto-play Narrative）： 提供一键播放功能，将静态数据转化为动态叙事，用户无需手动操作即可观察 2014-2021 年的全球排名大洗牌。
    • 欺骗性切换（Deceptive Mode）： 响应课程要求，设计了“媒体夸大模式”。在该模式下，系统会自动执行坐标轴截断（Truncated Y-Axis），将微小的波动渲染成剧烈的突变，展示了可视化作为叙事工具的危险性。
    • 即时洞察卡片（Insight Card）： 实现了自动化文本生成，实时计算选中时段的变动率，从“数据展示”跨越到“数据解读”。
  • 替代方案与废弃原因：最初考虑过使用“世界地图”，但由于本数据集包含许多面积微小但人均排放极高的国家（如卡塔尔、科威特），在地图上几乎不可见，会造成严重的视觉偏见。因此，最终决定采用排序条形图来保证每个国家都有平等的“发声权”。

3. 外部资源引用

  • 数据源： 数据来自 OECD (经济合作与发展组织) 环境数据库：Air and Greenhouse Gas Emissions (KG_CO2E_PS)。数据已清洗并处理为 JSON 格式。
  • 技术库： 使用了 Apache ECharts 实现底层图形渲染与动画，使用原生 JavaScript 处理多视图状态同步。

4. 开发流程概述

  • 总耗时： 约 22 个工时。
  • 时间分配：
    • 数据探索与清洗格式化：4小时（主要处理 OECD 的 SDMX 格式转 JSON，并提取各国人均值）。
    • 多视图联动逻辑开发：6小时（处理点击、滑动条与自动播放的同步状态）。
    • 欺骗性设计逻辑： 4小时（设计两套坐标轴动态计算算法）。
    • UI 精细化打磨与响应式适配：5小时（采用卡片式布局与 Inter 字体，优化移动端体验）。
    • 文档撰写与部署：3小时。
  • 开发评述： 在开发过程中，最核心的挑战在于“多视图协调（CMV）”的深度。当用户在“自动播放”过程中突然点击某个国家，系统需要同时处理时间轴推进、排名重绘以及特定国家的历史轨迹高亮。通过引入统一的状态管理逻辑，我们成功实现了引人入胜且逻辑严密的探索体验。
